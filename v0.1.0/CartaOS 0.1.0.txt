*** Para nossa versão 0.1.0:

*** Implementação para o Release 0.1.0:
Identificação: O sistema automaticamente move os arquivos que precisam de correção para a pasta 03_Lab.
Visualização na GUI: Em vez de apenas rodar um script, sua GUI deve ter uma área (uma aba ou seção "Laboratório") que lista os arquivos presentes no diretório 03_Lab.
Ação do Usuário: Para cada arquivo na lista, a GUI poderia oferecer um botão "Corrigir no ScanTailor". Clicar nesse botão abriria a ferramenta externa com o projeto do arquivo selecionado.
Finalização: Após você salvar seu trabalho no ScanTailor, a GUI precisaria de uma ação para finalizar a etapa, como um botão "Concluído, Mover para OCR". Esse comando então moveria o resultado do ScanTailor para a pasta 04_ReadyForOCR.
Isso transforma a etapa manual em uma tarefa gerenciada pelo CartaOS, o que é um objetivo muito mais realista e valioso para a primeira versão.

3. Empacotamento e Distribuição
Um "release" implica em ter um instalador ou um executável que outras pessoas (ou você mesmo, em outra máquina) possam usar facilmente.
Aproveite o Tauri: Você já está usando Tauri, que é perfeito para isso. Planeje usar o comando tauri build para gerar os pacotes de instalação para Windows, macOS e Linux. O seu arquivo tauri.conf.json já tem as configurações de ícone e de bundle preparadas para isso.

Riscos e mitigação rápidos
Dependências externas quebrando (ocr/tesseract): dry-run abrangente, mensagens de correção e checks no installer.
PDFs “sujos”: warnings capturados e bloco de “document quality warning” (você já fez isso — ótimo).
Falhas de API: retry simples com backoff e mensagem clara, fallback para debug.

Nice-to-have (0.2+)

Retentativa com backoff para OCR/IA.
Métricas simples e relatório final do batch.
CLI dry-run do pipeline completo.
Configuração externa (config.yaml) para linguagens OCR e paths.
Arquivo de configuração externo (config.yaml) para que usuários possam personalizar pacotes, linguagens OCR, etc.
Modo interativo com menus (mesmo no terminal), usando algo como questionary ou InquirerPy.
Fallback inteligente para quando algum pacote não estiver no package manager detectado (ex.: baixar binário oficial).
